# MoneyPop_CirclePop PRD

## 1. 문서 개요

### 1-1. 제품명
**MoneyPop_CirclePop**

### 1-2. 한 줄 설명
아래에서 위로 원형 도형을 밀어 올려 같은 단계끼리 합치고, 현재 목표 도형을 완성해 제거하며 점수를 얻는 **미니앱용 역중력 머지 퍼즐 게임**

### 1-3. 문서 목적
이 문서는 MoneyPop_CirclePop의 **확정된 핵심 규칙**, **프로토타입 구현 범위**, **예외 처리 방식**을 정의한다.  
개발자는 이 문서를 기준으로 별도 해석 없이 구현을 시작할 수 있어야 한다.

---

## 2. 제품 목표

### 2-1. 목표
이 게임의 목표는 아래 3가지다.

1. 유저가 **“내가 잘 배치해서 풀었다”**는 감각을 느끼게 한다.
2. 전투 없이도 합체와 제거 순간의 연출로 **시원한 타격감**을 제공한다.
3. 미니앱 환경에서 **짧고 반복 가능한 세션 구조**를 만든다.

### 2-2. 지향하는 플레이 감각
- 고난도 전략 게임이나 피지컬 게임이 아니다.
- 2048 수준의 **가벼운 선행 판단**을 유도한다.
- 실시간 반응속도보다 **배치 판단**이 중요하다.
- 운보다 실수와 선택이 더 잘 보이는 게임이어야 한다.

### 2-3. 비목표
프로토타입 단계에서는 아래를 목표로 하지 않는다.

- 다중 모드
- 고난도 메타 전략
- 대규모 아트 리소스
- 복잡한 성장 시스템
- 랭킹 경쟁 중심 구조

## 3. 핵심 결정 사항 요약

### 3-1. 확정된 핵심 규칙
- **조작 방식**: 드래그 앤 드롭 기반, 단 **아래에서 위로 직선 발사**
- **필드 구조**: 세로가 긴 자유 좌표 직사각형
- **벽 구조**: 상단/좌측/우측은 벽, 하단은 오픈
- **도형 형태**: 원형
- **단계 수**: 1~7단계
- **현재 도형 스폰 범위**: 1~3단계
- **현재 도형 스폰 가중치**: 6:3:1
- **목표 도형 범위**: 5~7단계
- **목표 도형 가중치**: 2:3:5
- **목표 개수**: 항상 1개
- **다음 도형 미리보기**: 제공하지 않음
- **합체 조건**: 같은 단계 도형이 충돌 또는 접촉하면 합체
- **연쇄 합체**: 허용
- **연쇄 중 목표 달성**: 즉시 처리
- **게임오버**: 새 도형을 유효하게 발사할 수 없는 상태가 되면 자동 판정

### 3-2. 플레이 철학
이 게임은 “실제로 어려운 게임”이 아니라,  
유저가 **살짝 똑똑한 선택을 했다고 느끼게 하는 게임**이어야 한다.

## 4. 핵심 플레이 루프

### 4-1. 기본 루프
1. 현재 목표 도형을 확인한다.
2. 현재 도형을 하단 발사 레일에서 좌우로 드래그한다.
3. 손을 놓으면 도형이 위로 직선 발사된다.
4. 도형은 역중력장 안에서 위쪽으로 이동한다.
5. 같은 단계 도형과 충돌하면 상위 단계로 합체한다.
6. 합체 결과가 현재 목표 도형이면 즉시 제거되고 점수를 얻는다.
7. 목표를 달성한 경우 새로운 목표 도형이 즉시 생성된다.
8. 연쇄 합체가 더 가능하면 계속 처리한다.
9. 필드가 안정화되면 다음 도형을 지급한다.
10. 다음 도형을 유효하게 넣을 수 없으면 게임오버다.

### 4-2. 핵심 재미 요소
- 배치 위치를 읽는 퍼즐성
- 연쇄 합체의 효율성
- 목표 도형 완성 시의 제거 쾌감
- 공간이 차오를수록 커지는 긴장감

## 5. 플레이 필드 정의

### 5-1. 필드 구조
플레이 필드는 **세로가 긴 직사각형**이며 내부는 **자유 좌표 기반**이다.  
그리드를 사용하지 않으며, 모든 도형은 원형 물리 바디로 존재한다.

### 5-2. 논리 좌표 기준
프로토타입 기준 논리 좌표는 아래와 같다.

- 필드 너비: **360**
- 필드 높이: **640**
- 좌측 벽 두께: **12**
- 우측 벽 두께: **12**
- 상단 벽 두께: **12**
- 하단: **벽 없음**

### 5-3. 벽 구조
- 상단: 고정 벽
- 좌측: 고정 벽
- 우측: 고정 벽
- 하단: 개방

### 5-4. 구조적 의미
- 하단은 **발사 진입 공간**
- 상단은 **도형이 몰리고 압축되는 공간**
- 좌우 벽은 도형이 화면 밖으로 퍼지는 것을 막는다

이 구조는 일반적인 낙하형 머지 게임과 반대로,  
아래에서 위로 도형이 쌓이며 압박이 증가하는 경험을 만든다.

## 6. 조작 방식

### 6-1. 입력 원칙
유저는 **발사 위치만 결정**한다.  
각도 조절, 힘 조절, 회전 조절은 제공하지 않는다.

### 6-2. 입력 방식
- 현재 도형은 필드 하단 바깥의 발사 레일 위에 표시된다.
- 유저는 도형을 좌우로 드래그할 수 있다.
- 손을 놓으면 해당 x좌표에서 **위로 직선 발사**된다.
- 발사 방향은 항상 동일하다.

### 6-3. 발사 가능 범위
도형 중심의 x좌표는 아래 범위에서만 선택 가능하다.

- 최소 x = `좌측 벽 두께 + 도형 반지름`
- 최대 x = `필드 너비 - 우측 벽 두께 - 도형 반지름`

### 6-4. 발사 시작 좌표
도형은 아래 위치에서 생성되어 필드 안으로 들어간다.

- 시작 x = 유저가 선택한 x
- 시작 y = `필드 높이 + 현재 도형 반지름 + 8`

### 6-5. 초기 물리값
프로토타입 초기값은 아래와 같다.

- 초기 속도: `(0, -10)`
- 월드 중력 방향: 위쪽
- 역중력 계수: `-0.55`
- 반발 계수: 낮게 설정
- 마찰: 중간 이하 수준

정확한 튜닝 수치는 개발 과정에서 조정 가능하지만,  
**항상 위로 직선 발사되는 구조는 변경하지 않는다.**

### 6-6. 유효하지 않은 발사
현재 선택한 x에서 도형을 생성했을 때 기존 도형과 겹쳐 **유효한 시작 상태를 만들 수 없으면**, 발사는 실행되지 않는다.

이 경우:
- 도형은 발사되지 않는다.
- 유저는 다른 x로 다시 드래그해야 한다.
- 단, 가능한 x가 하나도 없으면 이 상태는 게임오버로 이어진다.

## 8. 현재 도형 스폰 규칙

### 8-1. 스폰 범위
매 턴 지급되는 현재 도형은 **1~3단계만 등장**한다.

### 8-2. 스폰 가중치
- 1단계: 6
- 2단계: 3
- 3단계: 1

### 8-3. 확률 해석
- 1단계: 60%
- 2단계: 30%
- 3단계: 10%

### 8-4. 지급 타이밍
다음 도형은 아래 조건이 모두 끝난 뒤 지급된다.

1. 현재 턴의 합체 처리 종료
2. 목표 달성/제거 처리 종료
3. 연쇄 합체 종료
4. 물리 상태 안정화 완료

### 8-5. 물리 안정화 기준
프로토타입 기준 안정화 조건은 아래와 같다.

- 필드 내 모든 도형의 속도 절댓값이 **0.15 이하**
- 대기 중인 합체 이벤트가 없음
- 목표 제거 애니메이션이 종료됨

## 9. 목표 도형 규칙

### 9-1. 목표 범위
현재 목표 도형은 **5~7단계 중 하나**다.

### 9-2. 목표 가중치
- 5단계: 2
- 6단계: 3
- 7단계: 5

### 9-3. 목표 개수
- 항상 하나만 활성화된다.
- 복수 목표는 없다.

### 9-4. 목표 생성 시점
- 게임 시작 시 1회 생성
- 현재 목표를 달성했을 때만 새 목표 생성
- 현재 목표를 달성하지 못하면 목표는 유지된다

### 9-5. 목표 달성 조건
합체 결과 새로 생성된 도형의 단계가 **현재 목표 단계와 정확히 일치하면**, 그 순간 목표 달성으로 처리한다.

### 9-6. 목표 달성 즉시 처리 순서
연쇄 중이라도 목표를 달성하면 아래 순서로 **즉시 처리**한다.

1. 합체 결과 도형 생성
2. 현재 목표와 일치 여부 확인
3. 일치하면 해당 도형을 즉시 필드에서 제거
4. 점수 지급
5. 새로운 목표 도형 즉시 생성
6. 남아 있는 도형들에 대해 연쇄 합체 처리를 계속 진행

즉, **연쇄가 끝날 때까지 목표 갱신을 미루지 않는다.**

### 9-7. 연쇄 중 새 목표 재달성 처리
연쇄가 아직 끝나지 않은 상태에서 새 목표가 생성되었고,  
그 뒤 이어지는 연쇄 안에서 다시 새 목표와 일치하는 도형이 만들어지면:

- 그것도 즉시 목표 달성으로 처리한다
- 해당 도형은 즉시 제거된다
- 다음 목표가 다시 즉시 생성된다
- 이후 연쇄 처리를 계속한다

즉, 한 번의 발사 안에서 **목표를 여러 번 연속 달성할 수 있다.**

### 9-8. 목표 7단계 등장 정책
프로토타입에서는 목표 7단계의 등장 가중치 5를 그대로 유지한다.  
이는 의도적으로 **긴장감과 중장기 목표감**을 주기 위한 결정이다.

추후 밸런싱 단계에서 조정 검토는 가능하지만,  
현재 버전에서는 별도의 점진 해금 규칙을 두지 않는다.

## 10. 합체 규칙

### 10-1. 합체 트리거
같은 단계 도형 두 개가 **충돌하거나 물리 흔들림으로 접촉 상태가 되면** 합체한다.

### 10-2. 판정 기준
합체 가능 조건은 아래와 같다.

- 두 도형의 단계가 동일
- 두 도형의 중심 거리 <= 두 반지름 합 + 허용 오차
- 허용 오차 = **2**

### 10-3. 적용 대상
합체는 방금 발사한 도형에만 적용되는 것이 아니다.  
필드 안의 모든 도형이 합체 대상이다.

즉 아래 상황이 모두 가능하다.

- 발사 도형 vs 기존 도형
- 기존 도형 vs 기존 도형
- 연쇄로 생성된 도형 vs 다른 도형

### 10-4. 같은 단계 충돌 시 처리
같은 단계 도형 A, B가 합체하면:

- A 제거
- B 제거
- 다음 단계 도형 1개 생성
- 생성 좌표 = `(A 중심 + B 중심) / 2`
- 생성 속도 = `(A 속도 + B 속도) / 2`

### 10-5. 다른 단계 충돌 시 처리
단계가 다르면 합체하지 않는다.  
각 도형은 물리 반응만 수행하며 계속 움직이거나 정착한다.

### 10-6. 최대 단계 규칙
최대 단계는 7단계다.

- 7단계끼리 추가 합체는 없다
- 8단계는 존재하지 않는다
- 7단계가 목표가 아니면 필드에 남는다

### 10-7. 7단계 적체 정책
목표가 아닐 때 7단계 도형이 필드에 남아 공간을 차지하는 것은 **의도된 긴장감**이다.  
이는 리스크가 아니라 게임의 압박 구조로 간주한다.

즉:
- 7단계는 제거 수단이 제한된 큰 장애물 역할도 한다
- 유저는 목표 단계와 필드 공간을 동시에 고려해야 한다

---

## 11. 연쇄 합체 처리 규칙

### 11-1. 연쇄 허용
한 번의 발사 이후, 합체가 더 이상 발생하지 않을 때까지 연쇄 합체를 계속 처리한다.

### 11-2. 기본 진행 원리
- 발사 도형은 아래에서 위로 직선 이동
- 첫 충돌 대상이 같은 단계면 즉시 합체
- 합체 결과물은 평균 좌표에서 생성
- 결과물은 계속 역중력과 충돌의 영향을 받음
- 이후 추가 접촉이 발생하면 다시 합체 가능

### 11-3. 기존 도형끼리의 연쇄
발사 도형이 직접 닿지 않아도,
충돌 여파로 기존 도형들끼리 접촉하면 그들끼리도 합체한다.

### 11-4. 동시 후보 처리 순서
동일 시점에 합체 후보가 여러 개면 아래 규칙으로 순차 처리한다.

1. 현재 프레임의 합체 후보 쌍을 수집
2. 충돌 발생 시간 기준 정렬
3. 시간이 같으면 평균 y가 더 작은 쌍을 먼저 처리
4. y도 같으면 x가 더 작은 쌍을 먼저 처리
5. 한 쌍을 처리한 뒤 상태를 재계산
6. 더 이상 후보가 없을 때까지 반복

### 11-5. 목표 달성 포함 연쇄 처리
연쇄 중 목표 도형이 생성되면 즉시 제거하고 새 목표를 생성한 뒤,  
남은 도형들에 대한 연쇄 처리를 계속한다.  
이 규칙은 연쇄 종료까지 반복 적용된다.

## 12. 게임오버 규칙

### 12-1. 게임오버 정의
현재 상태에서 **새 도형을 유효하게 발사할 수 있는 x좌표가 하나도 없으면** 게임오버다.

### 12-2. 검사 시점
게임오버 검사는 아래 시점에 자동으로 수행한다.

1. 현재 턴의 연쇄 합체 종료 후
2. 물리 상태 안정화 후
3. 다음 도형이 결정된 직후

즉, 다음 도형이 정해졌을 때 **즉시 자동 판정**한다.

### 12-3. 유효 발사 가능 여부 검사
현재 지급된 도형의 반지름을 기준으로,
발사 레일 범위 안에서 생성 좌표가 기존 도형과 겹치지 않는 x가 하나라도 있는지 검사한다.

- 하나라도 있으면 게임 계속
- 하나도 없으면 게임오버

### 12-4. 6-6과의 관계
- 유효한 x가 **하나 이상 존재**할 때:  
  유저가 막힌 위치에 두면 발사 실패 후 다시 드래그 가능
- 유효한 x가 **하나도 존재하지 않을 때**:  
  자동으로 게임오버 처리

즉, 발사 실패와 게임오버는 서로 다른 상황이다.

### 12-5. 게임오버 처리 순서
게임오버가 되면 아래 순서로 처리한다.

1. 최종 게임 상태 저장
2. 게임 입력 비활성화
3. `GAME OVER` 메시지 표시
4. `다시 하기` 버튼 표시

### 12-6. 다시 하기
유저가 `다시 하기` 버튼을 누르면:

1. 새로운 게임 세션 시작
2. 점수 초기화
3. 필드 초기화
4. 새 목표 생성
5. 새 현재 도형 생성

## 13. 점수 규칙

### 13-1. 점수 원칙
점수는 **목표 도형 달성 시에만** 획득한다.

### 13-2. 단계별 점수
| 목표 단계 | 점수 |
|---|---|
| 5단계 | 100점 |
| 6단계 | 250점 |
| 7단계 | 600점 |

### 13-3. 다중 목표 달성
한 번의 발사와 연쇄 안에서 목표를 여러 번 달성할 수 있으며,  
그 경우 매번 해당 점수를 누적 지급한다.

### 13-4. 콤보 점수
프로토타입에서는 별도 콤보 점수는 없다.

## 14. UX 및 화면 구성

### 14-1. 메인 화면
- 상단: 현재 점수, 현재 목표 도형, 설정/일시정지 버튼
- 중앙: 세로형 플레이 필드
- 하단: 현재 도형, 발사 레일

### 14-2. 다음 도형 미리보기
**제공하지 않는다.**

설계 이유:
- 운 요소를 약간 남겨 과도한 계산 게임이 되지 않도록 하기 위함
- 미니앱 환경에서 화면 밀도를 낮추기 위함
- 핵심 판단을 “현재 한 수의 배치”에 집중시키기 위함

### 14-3. 목표 도형 표시
현재 목표는 상단에 항상 크게 노출한다.

표시 요소:
- 목표 단계 색상
- 목표 원형 아이콘
- 짧은 텍스트 라벨

### 14-4. 발사 피드백
- 유효 위치: 정상 표시
- 유효하지 않은 위치: 경고 표시
- 발사 시: 위쪽 방향성 강조 이펙트
- 목표 달성 시: 강한 제거 연출

### 14-5. 게임오버 화면
표시 항목:
- GAME OVER 메시지
- 최종 점수
- 이번 세션 최고 달성 단계
- 다시 하기 버튼

## 19. 리스크 및 대응

### 19-1. 너무 운빨처럼 느껴질 위험
#### 대응
- 각도/힘 조절 제거
- 발사 위치만 판단하게 설계
- 스폰 단계를 1~3으로 제한

### 19-2. 물리 흔들림이 억울하게 느껴질 위험
#### 대응
- 낮은 반발 계수 사용
- 허용 오차를 제한
- 발사 속도와 역중력 강도를 과도하지 않게 유지

### 19-3. 목표 7단계가 초반부터 어렵게 느껴질 위험
#### 대응
- 프로토타입에서는 의도된 긴장감으로 수용
- 도형 반지름 축소로 플레이 지속 시간을 확보
- 추후 테스트 결과에 따라 가중치 조정 여부 검토 가능

### 19-4. 7단계 적체가 과도한 압박이 될 위험
#### 대응
- 현재 버전에서는 의도된 긴장감으로 간주
- 공간 압박 자체를 핵심 재미의 일부로 본다

### 19-5. 하단 오픈 구조가 직관적이지 않을 위험
#### 대응
- 발사 레일과 위쪽 방향 가이드를 명확히 노출
- 첫 플레이에서 짧은 시각 튜토리얼 제공 가능

## 20. 최종 요약

MoneyPop_CirclePop은 **아래에서 위로 발사하는 역중력 머지 퍼즐 게임**이다.

핵심 규칙은 다음과 같다.

- 세로형 자유 좌표 필드
- 상단/좌우 벽, 하단 오픈
- 현재 도형은 1~3단계만 스폰
- 목표 도형은 5~7단계 중 1개
- 유저는 하단 레일에서 현재 도형을 좌우로 이동 후 위로 직선 발사
- 같은 단계 원형 도형이 충돌/접촉하면 합체
- 합체 결과가 현재 목표면 즉시 제거 및 점수 획득
- 연쇄 중 새 목표가 다시 달성되어도 즉시 처리
- 다음 도형 미리보기는 없음
- 새 도형을 유효하게 넣을 x가 하나도 없으면 자동 게임오버
- 게임오버 시 최종 상태 저장 후 다시 하기 제공

이 문서를 기준으로 프로토타입 개발을 진행한다.
